# Vector Databases

A vector database is a specialized data store designed for indexing, storing, and querying high-dimensional vector embeddings generated by machine learning models.

## Core Concepts

**Embeddings** are dense numerical representations of text, images, or other data. An embedding model converts input data into a fixed-length vector (e.g., 768 or 1536 dimensions) that captures semantic meaning.

**Similarity search** finds vectors closest to a query vector using distance metrics like cosine similarity, Euclidean distance, or dot product.

**Approximate Nearest Neighbor (ANN)** algorithms like HNSW, IVF, and ScaNN enable sub-millisecond search across billions of vectors by trading exact accuracy for speed.

## Databricks Vector Search

Databricks Vector Search is a serverless vector database built into the Databricks platform. Key features:

- **Delta Sync** — Automatically syncs embeddings from Delta tables, keeping the index current as source data changes.
- **Managed embeddings** — Can generate embeddings automatically using the [[foundation-model-api]] endpoint `databricks-gte-large-en`.
- **Hybrid search** — Combines vector similarity with keyword filtering for more precise results.
- **[[unity-catalog]] integration** — Vector search indexes are governed by Unity Catalog permissions.

## Role in the LLM Wiki

In the [[llm-wiki-pattern]], vector search provides the semantic retrieval layer for the query operation. When a user asks a question, the system:

1. Performs full-text search for exact keyword matches
2. Performs vector similarity search for semantically related pages
3. Merges results using reciprocal rank fusion (RRF)

This hybrid approach ensures both precise and exploratory results. The vector index is built on compiled wiki pages rather than raw source chunks, which gives better retrieval quality since pages contain synthesized, cross-referenced content.
